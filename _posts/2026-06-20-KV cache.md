>[!最重要前提]
>KV Cache是指存储旧token的kv矩阵，所谓MQA\GQA\MLA是对“存储”的数据的**剪枝**操作
>如果问：KV Cache时间复杂度节省了多少？KV Cache 把**单步解码复杂度从平方 O(n2) 降到线性 O(n)**
>分析：$输入长度为N的d维token序列，原时间复杂度：N个qkv互相都要计算：[qk^T=>O(N^2d)] + [softmax(scorces)*v=>O(N^2d) ] ====>>>> O(2*N^2*d)$
> $kv cahe节省的是 旧token的kv的计算，只进行新的token计算，计算新token与旧N-1token所以现在的时间复杂度query_new*kv + softmax(scores_new)*v =>O(2N*d)$



在推理过程中，由于模型堆叠了多层transformer，所以核心的计算消耗在Transformer内部，包括MHA，FFN等操作，其中MHA要计算Q，K ，V 矩阵，来做多头注意力的计算。前序token序列计算的K V 结果，会随着后面token推理过程逐步存到显存里。存储的量随着Batch，Sequence_len长度动态变化。
# 1. 基础MHA和KV cache必要性

已知transformer的多头注意力机制MHA会对每一个token生成qkv，
假设输入prompt是4个token：[A, B, C, D]，模型要生成3个续写token [X, Y, Z]。
**自回归生成：** 自回归生成是**逐个token生成**，每次生成的token会**拼接到输入末尾**，作为下一步的输入。
**Step1：**[A, B, C, D]每个 token 映射为$x_1,x_2,x_3,x_4$，然后各自通过线性层产生自己的 Q、K、V，然后所有 token 的 K、V 被每个 token 的 Q 查询得到注意力输出向量，FFN-MLP-softmax-argmax->c1

下面我复习以下mha具体过程：
	具体过程举例：得到q_2对于所有kv的**注意力分数向量atten**，长度是输入token 序列长度$$atten[i] = softmax(q*k_i^T/sqrt(d_k))$$加权求和得到每个位置的注意力输出$$out\_atten\_2 = \sum{atten[i]*v_i}$$所以每个位置都会得到一个注意力输出向量 $out_attn_i$
	然后，Add & Norm（残差连接 + LayerNorm），得到$x_1$在第一个block对应的输出：$$x_1(1) = LayerNorm( x_1(0) + out_attn_1 )$$
	整个输出就是$[x_1(1),x_2(1),x_3(1),x_4(1)]$，它的维度<=>$[x_1,x_2,x_3,x_4]$
	经过多个block之后，假设4个block，得到$[x_1(4),x_2(4),x_3(4),x_4(4)]$
	**只取最后一个位置的输出**$x_4(4)$，经过MLP映射到词表大小$logits = x_4(4) * W_out + b_out$，再经过softmax得到概率分布$probs = softmax(logits)$
	为什么只取最后一个位置的输出$x_4(4)$？$[x_1,x_2,x_3,x_4]$所有信息已经通过因果注意力和层间传递，汇聚到了最后一个 token 的表示中
	对于$probs = softmax(logits)$，需要从中选取一个概率对应的下标作为生成的token索引，如何选：argmax、beam search等。
**Step2：** **将$x_1,x_2,x_3,x_4,c_1$作为输入**，继续计算$c_2$，这个过程**还需要用到$x_1,x_2,x_3,x_4$的 kv，新 token 必须与所有历史 token 的 K、V 计算注意力。**
所以将旧token的kv缓存下来，避免重复计算带来的时间、资源浪费。

为什么不缓存Q：类似于在做一份长长的笔记，每写一个新句子，你需要回顾之前所有句子的**内容**（相当于 K 和 V），但是你不必重新思考“我之前写第一个句子时是怎么想的”（相当于 Q）。你只需要**用当前 Q(c1)去匹配之前token的（缓存 KV），所以Q 只在每个 token 第一次出现时计算一次，不需要缓存Q。**


“**注意力压缩**”指为了解决 Transformer 模型中标准注意力机制的平方级复杂度O(N²)问题而发展起来的一系列优化技术。简而言之，降低MHA的时间复杂度O(N²)。
目前有几种技术，在使用时经常进行**多策略融合**，动态组合多种技术，例如使用 **GQA 减少内存占用，结合 FlashAttention 加速计算，再引入 KV Cache 剪枝策略进一步优化显存**，以实现综合最佳效果：
- **共享KV**：多个Head共享使用1组KV，将原来每个Head一个KV，变成1组Head一个KV，来压缩KV的存储。代表方法：GQA，MQA等
- **窗口KV**：针对长序列控制一个计算KV的窗口，KV cache只保存窗口内的结果（窗口长度远小于序列长度），超出窗口的KV会被丢弃，通过这种方法能减少KV的存储，当然也会损失一定的长文推理效果。代表方法：Longformer等
- **量化压缩**：基于量化的方法，通过更低的Bit位来保存KV，将单KV结果进一步压缩，代表方法：INT8等
- **计算优化**：通过优化计算过程，减少访存换入换出的次数，让更多计算在片上存储SRAM进行，以提升推理性能，代表方法：flashAttention等



# 2. MQA（Multi-Query Attention）
从x经过三个线性层映射为QKV时，三个线性层的维度并不一致，也就是分头之前Q和KV的维度不一致。
设置头num_head，token embedding(x)维度d_model，x经过三个线性层映射得到的Q、K、V，Q的维度是d_model然后均分为num_head份，每一份是d_k=d_model/num_head；K、V的维度直接就是d_k
**k** 和 **v** 会被**所有Q 头共享**。

# 3.GQA
对于MQA，所有 Q 头共享完全相同的 KV，可能损失表达能力，影响模型质量（尤其是复杂任务），所以多设几组kv头来缓解这种情况。
比如设置g组，将num_head个Q头分成g组，每一组共享一组kv
那么k、v的线性层的输出维度应该是$d_k*g$，MQA就可以看做GQA的g=1的特例



# MLA：多头潜在注意力(Multi-Head Latent Attention)

*三句话总结：低秩压缩KV，将KV压缩到一个低秩矩阵C，使用时解压缩。因为经过压缩解压缩，所以原本的位置编码RoPE不能用了，需要解耦操作，最后对于Q也有一些处理。需要额外训练“压缩”“解压缩”的投影层）*

与参数共享的思路不同，采用 **“低秩压缩”** 思想，不再为每个Token存储完整的KV信息，而是存储其低维、压缩的“潜在表示”（latent representation），在计算时才动态、实时地“按需解开”，还原出KV矩阵
DeepSeek描述自己的方法不仅比MQA强，而且比非共享KV的原始MHA也要强
**低秩（Low-Rank）联合压缩 + 实时上投影**（Up-Projection）

共享K、V，引入两个**低秩变换矩阵**、key：$压缩矩阵W_D (down)、 解压矩阵W_U (up)$

解压后的q,k由两部分组成$q=[q_C,q_R],k=v=[k_C,k_R]$

流程：
压缩、缓存：$x \rightarrow C_{KV}=x*W_D  ，缓存时只缓存向量C$
解压：$C_{KV} \rightarrow KV = C_{KV}·W_U$


“Latent潜在”：KV矩阵**压缩**到低维潜在空间，仅存储压缩后的潜向量，推理时**动态重构**完整KV
```
第一次计算kv矩阵->压缩存储
复用推理时动态重构
```
